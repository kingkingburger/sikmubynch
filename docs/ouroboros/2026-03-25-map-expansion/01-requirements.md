# Phase 1: 요구사항 — 대규모 맵 확장

## 목표 (Goal)

64x64 맵을 **256x256**으로 확장하여 They Are Billions 스타일 **거점 확장형** 웨이브 디펜스를 구현한다. 플레이어는 HQ 주변 소규모 영역에서 시작하여 바리케이드/타워로 영역을 넓혀가며, 맵 곳곳의 자원 지대를 확보한다.

**한 문장 목표:** "안개 속에서 시작하여 영역을 넓혀가며 대규모 웨이브를 방어하는 거점 확장형 전투"

## 핵심 엔티티 (Ontology)

| 엔티티 | 설명 |
|--------|------|
| **Territory** | 플레이어가 점령한 영역. 바리케이드/건물 배치 시 확장됨 |
| **Fog of War** | 미탐색 영역은 어둡게 처리. 건물/유닛 시야 범위만 보임 |
| **Resource Zone** | 맵에 산재한 미네랄 풍부 지역. 채굴기 설치로 수입 증가 |
| **Spawn Point** | 맵 4방향 경계 + 중간 거점에서 적 출현 |
| **Minimap** | 256x256 맵 전체를 축소 표시하는 UI 요소 |
| **Chunk** | FlowField 계산 단위. 맵을 32x32 청크 8x8로 분할 |

## 제약조건 (Constraints)

1. **성능**: 256x256 맵에서 1000+ 적 동시 처리, 60fps 유지
2. **FlowField**: 전체 BFS 불가 → 청크 기반 또는 계층적 길찾기 필수
3. **SpatialGrid**: 셀 크기/개수 비례 확장 (8.0 셀 × 32x32 그리드)
4. **카메라**: 줌 아웃 최대 120, 미니맵 필수
5. **GL Compatibility**: 렌더러 제약 유지 (포그 오브 워는 셰이더 기반)
6. **메모리**: FlowField Dictionary 256x256 = 65,536 엔트리 (관리 가능)
7. **기존 게임 루프**: 웨이브 → 건설 → 보상 카드 사이클 유지

## 비목표 (Non-Goals)

1. 절차적 맵 생성 (런마다 랜덤 맵) — 이번 스코프 아님
2. 멀티플레이어 — PC 싱글 전용 유지
3. 유닛 AI 완전 재설계 — 기존 순찰/추적/공격 유지, 범위만 확장
4. 3D 모델 교체 — 별도 작업 (이미 Ouroboros 문서 있음)

## 수용 기준 (Acceptance Criteria)

| # | 기준 | 검증 방법 |
|---|------|-----------|
| AC1 | 맵 크기 256x256, 건물 배치 전체 영역 가능 | 맵 경계 4곳에 건물 배치 확인 |
| AC2 | FlowField 재계산 < 100ms (256x256) | 타이머 로그 |
| AC3 | 1000 적 동시, 60fps 이상 | F3 디버그 오버레이 확인 |
| AC4 | 안개: 건물/유닛 시야 외 영역은 어둡게 | 시각적 확인 |
| AC5 | 미니맵에 건물/적/자원지대 표시 | UI 확인 |
| AC6 | 자원 지대 최소 8곳, 채굴기 설치로 수입 증가 | 게임플레이 확인 |
| AC7 | 적 스폰: 4방향 맵 경계 + 웨이브별 방향 변화 | 관찰 |
| AC8 | 카메라 줌/팬이 256x256 전체 커버 | WASD + 줌 테스트 |

## 가정과 해결 (Assumptions)

| 가정 | 해결/결론 |
|------|-----------|
| 256x256 BFS가 너무 느림 | 청크 기반 2단계 길찾기로 해결 (글로벌 청크→로컬 셀) |
| 안개가 성능에 영향 | 셰이더 기반 (visibility 텍스처), CPU 비용 거의 없음 |
| 1000+ 적 move_and_collide 비용 | SpatialGrid 확장 + 적끼리 충돌 무시로 해결 |
| 자원 지대 배치 밸런스 | 고정 위치 (런마다 동일, 밸런스 수동 조정) |
| HQ 위치 | 맵 중앙 (128, 128) |
