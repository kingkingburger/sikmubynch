# SIKMUBYNCH (식무변처) - 검증

> Ouroboros Phase 3 | 2026-03-19 | 모호성: 16%

## 검증 전략

**병렬 검증** — 성능, 게임 필, 시스템 정합성을 초기 프로토타입부터 동시에 점진적 검증.

| 카테고리 | 검증 방법 | 자동화 |
|---------|----------|--------|
| 성능 | 벤치마크 스크립트 | 자동 |
| 게임 필 | 직접 플레이테스트 | 수동 (체크리스트) |
| 시스템 정합성 | GdUnit4 단위 테스트 + 수동 확인 | 반자동 |

## 1. 성능 검증 시나리오

### S-PERF-1: 기본 대규모 전투

- **전제 조건**: 게임 실행 중, 웨이브 진행
- **행동**: 적 500마리 동시 존재 + 아군 유닛 50마리 + 건물 20개 + 발사체/파티클 활성
- **기대 결과**: 60fps 이상 유지
- **검증 방법**: `Engine.get_frames_per_second()` 로깅, 10초간 평균/최소 fps 측정
- **성공 기준**: 평균 60fps 이상, 최소 45fps 이상 (순간 드롭 허용)
- **자동화**: 벤치마크 씬 (`benchmark_500.tscn`) — 적 500마리 자동 스폰 + fps 로그

### S-PERF-2: 극한 상황

- **전제 조건**: 후반 웨이브 (Wave 30+)
- **행동**: 적 1000마리 동시 존재 + 대량 파티클
- **기대 결과**: 30fps 이상 유지
- **검증 방법**: 위와 동일
- **성공 기준**: 평균 30fps 이상 (플레이 가능 수준)
- **자동화**: 벤치마크 씬 (`benchmark_1000.tscn`)

### S-PERF-3: Flow Field 재계산

- **전제 조건**: 적 500마리 이동 중
- **행동**: 건물 배치 → Flow Field 재계산 트리거
- **기대 결과**: 재계산 중 프레임 드롭 없음 (16ms 이내 완료)
- **검증 방법**: `Time.get_ticks_msec()` 전후 측정
- **성공 기준**: Flow Field 재계산 < 16ms (64×64 그리드)
- **자동화**: 유닛 테스트로 재계산 시간 측정

### S-PERF-4: 메모리 사용

- **전제 조건**: 장시간 플레이 (30웨이브+)
- **행동**: 적 반복 생성/사망/풀 반환
- **기대 결과**: 메모리 누수 없음
- **검증 방법**: `OS.get_static_memory_usage()` 주기적 로깅
- **성공 기준**: 메모리 증가율 < 1MB/분
- **자동화**: 30웨이브 자동 플레이 스크립트 + 메모리 로그

## 2. 게임 필 검증 시나리오 (직접 플레이테스트)

### S-FEEL-1: 타격감 — 일반 적 처치

- **전제 조건**: 타워/유닛이 러셔를 공격
- **행동**: 적 1마리 사망 관찰
- **체크리스트**:
  - [ ] 피격 시 적 스프라이트 white flash 보임
  - [ ] 사망 시 파티클 이펙트 발생
  - [ ] 사망 시 약한 화면 흔들림 느껴짐
  - [ ] 적 사망 사운드 재생
  - [ ] "기분 좋다"는 느낌이 드는가? (주관)

### S-FEEL-2: 타격감 — 대량 적 처치

- **전제 조건**: 바리케이드 앞에 적 30마리+ 밀집
- **행동**: 범위 공격으로 다수 동시 처치
- **체크리스트**:
  - [ ] 동시 다수 사망 시 강한 화면 흔들림
  - [ ] 파티클이 화면을 채우는 쾌감
  - [ ] 히트스톱(순간 슬로모션) 체감
  - [ ] "시원하다" / "쾌감" 느낌이 드는가? (주관)

### S-FEEL-3: 타격감 — 대형 적 처치

- **전제 조건**: 탱크형 적 1마리 존재
- **행동**: 장시간 공격 후 사망
- **체크리스트**:
  - [ ] 처치 시 강한 화면 흔들림 + 히트스톱
  - [ ] 큰 폭발 파티클
  - [ ] "해냈다" 성취감이 드는가? (주관)

### S-FEEL-4: 긴장감 — 1웨이브

- **전제 조건**: 런 시작 직후
- **행동**: 첫 웨이브 45마리 러셔 밀려옴
- **체크리스트**:
  - [ ] 시작하자마자 "많다" 느낌
  - [ ] 빠르게 건설하지 않으면 위험하다는 압박감
  - [ ] 방심하면 실제로 죽을 수 있는가?
  - [ ] "긴장된다"는 느낌이 드는가? (주관)

### S-FEEL-5: 리플레이 — "한 판 더"

- **전제 조건**: 3판 연속 플레이 후 사망
- **체크리스트**:
  - [ ] 죽었을 때 "이번엔 다른 전략으로" 생각이 드는가?
  - [ ] 시너지 조합 때문에 매 판이 달랐는가?
  - [ ] 랜덤 맵/보상으로 같은 전략이 통하지 않았는가?
  - [ ] 즉시 다시 시작 버튼을 누르고 싶은가? (핵심)

### S-FEEL-6: 대규모 전투 스펙터클

- **전제 조건**: Wave 10+ 도달
- **행동**: 200마리+ 적이 밀려오는 장면 관찰
- **체크리스트**:
  - [ ] 적 물결이 시각적으로 압도적인가?
  - [ ] 방어선과 충돌하는 순간이 극적인가?
  - [ ] "와 이거 대단하다"는 느낌이 드는가? (주관)

## 3. 시스템 정합성 검증

### S-SYS-1: 시너지 발동/해제

- **전제 조건**: 빈 맵, 자원 충분
- **행동**: 화염 특성 건물 1개 + 화염 특성 유닛 2마리 배치
- **기대 결과**: 화염폭풍 시너지 발동 → UI 바 업데이트 + 필드 오라
- **후속**: 화염 건물 제거 → 시너지 해제 확인
- **성공 기준**: 시너지 발동/해제가 즉시 반영, UI와 필드 이펙트 동기화
- **자동화**: GdUnit4 테스트 — `SynergyManager.get_active_synergies()` 검증

### S-SYS-2: 보상 3택

- **전제 조건**: 웨이브 클리어 직전
- **행동**: 마지막 적 처치 → 보상 화면
- **기대 결과**:
  - 게임 일시정지
  - 3개 보상 표시 (중복 없음)
  - 희귀도에 따른 시각적 차이
  - 선택 즉시 적용 → 게임 재개
- **성공 기준**: 보상 적용 후 즉시 효과 체감 가능
- **자동화**: 보상 생성 로직 단위 테스트 (중복 검증, 희귀도 분포)

### S-SYS-3: 웨이브 스케일링

- **전제 조건**: 자동 플레이 모드
- **행동**: Wave 1~20 자동 진행
- **기대 결과**:
  - 매 웨이브 적 수 증가
  - 적 종류 순차 등장 (러셔→탱크→스플리터→폭발형)
  - 체력/속도 배수 올바르게 적용
- **성공 기준**: 설계 문서의 수식과 일치
- **자동화**: GdUnit4 테스트 — `WaveManager.get_wave_config(n)` 값 검증

### S-SYS-4: Flow Field 경로

- **전제 조건**: 건물로 미로 형태 구성
- **행동**: 적 스폰 → 이동 관찰
- **기대 결과**: 적이 건물을 피해 본진으로 이동
- **검증**: 적이 건물 통과하지 않음, 막힌 경우 가장 가까운 건물 공격
- **자동화**: 유닛 테스트 — 알려진 맵 레이아웃에서 경로 검증

### S-SYS-5: 건물 배치 유효성

- **전제 조건**: 건설 모드 활성
- **행동**: 다양한 위치에 건물 배치 시도
- **기대 결과**:
  - 기존 건물 위: 배치 불가 (빨간 표시)
  - 적 스폰 경로 완전 차단: 배치 불가 (적 진입 경로 최소 1개 보장)
  - 자원 부족: 배치 불가 (UI 비활성화)
  - 유효 위치: 배치 성공 (그리드 스냅)
- **자동화**: 부분 자동화 가능 (`BuildingPlacer.can_place()` 테스트)

### S-SYS-6: 런 리셋

- **전제 조건**: 게임 진행 중 (건물/유닛/시너지 활성)
- **행동**: 본진 파괴 → 게임오버 → 다시 시작
- **기대 결과**:
  - 모든 상태 초기화 (건물, 유닛, 적, 자원, 시너지, 웨이브)
  - 새 랜덤 맵 생성
  - 메모리 누수 없음
- **성공 기준**: 새 런이 첫 런과 동일한 초기 상태
- **자동화**: 10회 연속 런 시작/사망 반복 → 메모리 + 상태 검증

### S-SYS-7: Object Pool 정합성

- **전제 조건**: 장시간 플레이
- **행동**: 적 반복 생성/사망 500회+
- **기대 결과**:
  - 풀 고갈 없음 (사망 적 반환 정상)
  - 유령 적 없음 (죽은 적이 화면에 남지 않음)
  - 재활용 적 데이터 오염 없음
- **자동화**: 스트레스 테스트 스크립트

## 4. 엣지케이스 검증

### S-EDGE-1: 적이 경로 없음

- **상황**: 건물로 모든 방향 막음
- **기대**: 적이 가장 가까운 건물 공격 (벽 부수고 진입)
- **검증**: 수동 테스트

### S-EDGE-2: 스플리터 연쇄 분열

- **상황**: 스플리터 50마리 동시 사망
- **기대**: 분열 후 총 100마리 (1회 제한). 풀 한계 내 처리
- **검증**: 자동 — 풀 크기 모니터링

### S-EDGE-3: 모든 생산 건물 파괴

- **상황**: 배럭 전부 파괴, 유닛 생산 불가
- **기대**: 다음 보상에서 배럭 우선 제공 (자동 조정)
- **검증**: 수동 테스트 — 보상 풀 편향 확인

### S-EDGE-4: 자원 0 + 건물 0

- **상황**: 본진 외 건물 없음, 자원 0
- **기대**: 본진 자체 약한 자동 공격으로 최소한의 대응. 적 처치로 자원 획득 가능
- **검증**: 수동 테스트

### S-EDGE-5: 동시 다수 이벤트

- **상황**: 시너지 발동 + 웨이브 클리어 + 대형 적 사망 동시
- **기대**: 모든 이벤트 정상 처리, 보상 UI 올바르게 표시
- **검증**: 수동 테스트

### S-EDGE-6: 맵 생성 실패

- **상황**: 절차적 생성 시 적 진입 경로 없는 맵 생성
- **기대**: 생성 후 경로 유효성 검증 → 실패 시 재생성
- **검증**: 자동 — 1000회 맵 생성 + 경로 존재 검증

## 7. 자동화 실행 계획

### 벤치마크 씬

```
tests/
├── benchmark_500.tscn      # 적 500마리 자동 스폰 + fps 로그
├── benchmark_1000.tscn     # 적 1000마리 극한 테스트
├── stress_pool.gd          # Object Pool 스트레스 테스트
├── stress_mapgen.gd        # 맵 생성 1000회 유효성 검증
└── unit/
    ├── test_synergy.gd     # 시너지 발동/해제 테스트
    ├── test_wave.gd        # 웨이브 스케일링 수식 검증
    ├── test_flowfield.gd   # Flow Field 경로 검증
    └── test_building.gd    # 건물 배치 유효성 테스트
```

### 플레이테스트 체크리스트 (수동)

플레이테스트 시 아래 체크리스트를 순서대로 확인:

1. 런 시작 → 1웨이브 긴장감 (S-FEEL-4)
2. 일반 적 처치 → 타격감 (S-FEEL-1)
3. 대량 적 처치 → 쾌감 (S-FEEL-2)
4. 대형 적 처치 → 성취감 (S-FEEL-3)
5. 시너지 빌드 → 전략적 깊이 (S-SYS-1)
6. 보상 선택 → 의미 있는 선택인가 (S-SYS-2)
7. Wave 10+ → 대규모 스펙터클 (S-FEEL-6)
8. 사망 → "한 판 더" 충동 (S-FEEL-5)
9. 3판 연속 → 매 판 다른 경험인가 (S-FEEL-5)

### 디버그 오버레이 (개발용)

게임 내 F3 키로 토글:
- 현재 fps
- 활성 적 수
- Object Pool 사용량/잔여
- Flow Field 시각화 (방향 화살표)
- 시너지 상태
- 메모리 사용량
